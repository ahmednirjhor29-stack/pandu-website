PANDU
Intelligent Computational Archaeological Workspace

Structured analysis for the study of evolving writing systems

Pandu is a research workspace for studying how ancient writing systems change shape across time, region, and material — turning scattered inscriptions into structured, comparable, AI-assisted data.

The Problem

Scripts don't disappear all at once. They erode, letter by letter.

A cuneiform wedge, a Brahmi curve, a Phoenician stroke — each one drifts slightly with every century, region, and surface it's carved or written on. Epigraphists have always tracked this by eye. Pandu tracks it geometrically, so the drift becomes measurable instead of anecdotal.

01

Fragmented records

Photographs, rubbings, and transcriptions of the same script sit in disconnected museum archives, personal folders, and unindexed papers — rarely in a form that can be compared side by side.

02

No shared geometry

There's no common language for describing how a letterform bends, simplifies, or rotates as it moves between scripts and periods — the kind of description a classifier could actually learn from.

03

Expertise doesn't scale

Reading an unfamiliar script well takes years of training. Pandu won't replace that expertise — it exists to extend it, by handling the first pass across large volumes of material.

The Approach

Geometry first, language models second.

Pandu treats a letterform as a shape before it treats it as a symbol. Contour analysis, Procrustes alignment, Fourier descriptors, Hu moments, and PCA give it a way to describe how a glyph is shaped — before any AI model is asked to guess what it means.

Local + Cloud, by choice

Run it offline, or bring your own model

Pandu talks to a local Ollama instance for fully offline analysis, or to Gemini, OpenAI, Anthropic, and Mistral when cloud reasoning is preferred — the researcher decides where their data goes, per session.

Structured, not just chatted

Every finding lands in a database

Transcriptions, translations, confidence scores, and letterform notes are written straight into a queryable library — not lost at the bottom of a chat window.

Inside the Workspace

A dark, instrument-panel interface built for long analysis sessions

The desktop application is organised into focused pages — Data Entry, Library, AI Analysis, AI Database, Train, and Script Analysis — each colour-coded so the researcher always knows which layer of the pipeline they're in.

Progress

From a single-purpose script viewer to a full workspace

Pandu is being built in the open, one working feature at a time — no step skipped, nothing shipped that hasn't been run against real attempts at real scripts.

PHASE 00

ScriptLens — the original concept

Started as a focused research tool for studying the geometric shape of historical letterforms and tracking their transformation across time. Python, PyQt6, SQLite, OpenCV, ReportLab, Matplotlib.

PHASE 01

Core data pipeline

Drag-and-drop image ingestion, a structured Library database, and a dark, instrument-panel UI designed for long sessions rather than quick glances.

PHASE 02

Renamed and re-scoped as Pandu

Evolved into an "Intelligent Computational Archaeological Workspace" — a broader home for analysis, AI-assisted transcription, and model training, not just single-image inspection.

PHASE 03

Multi-provider AI + embedded chat

Added support for Gemini, OpenAI, Anthropic, and Mistral alongside local Ollama models, each with its own stored key and session settings — plus an inline chat interface for working through findings conversationally.

PHASE 04

AI Database, DB Query mode, and Training page

Every AI finding now lands in its own queryable database, separate from the source Library, with a dedicated Train page for running script-pattern analysis passes over selected records.

PHASE 05

Script Analysis + PDF reporting

A dedicated drag-and-drop analysis page that returns a structured translation, probable attributes, a statistical confidence chart, and a full PDF report per artifact.

NOW

Windows support & the dark monochrome restyle

Currently converting Pandu to run cleanly on Windows and finishing a full UI restyle to a consistent dark monochrome theme with colour-coded borders — the same theme this website borrows.

DOES MY DATA GET USED IMMEDIATELY?

04 Jul 2026UI / Windows

Restyling Pandu into a dark monochrome instrument panel

Why the interface is moving away from mixed accent colours toward a disciplined dark monochrome base with colour-coded borders — and what that's forcing me to fix along the way.

Read full post + comments ↓

The earlier interface worked, but it didn't yet look like a single tool — different pages had drifted into different visual habits. The current pass standardises everything onto one dark base (#070707 background, #0d0d0d cards) and uses border colour, not background colour, to signal what a panel is for: rust for user actions, cyan for AI output, moss for success states, violet for reporting.

Alongside the restyle, I'm working through the Windows port — mostly path handling and a couple of Qt behaviours that differ from Linux Mint, where the app has lived so far.

What's next

Finishing the restyle across the Train and Script Analysis pages, then a proper pass at packaging for Windows so it's not just "runs on my machine."

21 Jun 2026AI Pipeline

Four cloud providers, one settings panel

Adding Gemini, OpenAI, Anthropic, and Mistral support alongside local Ollama models — and why per-provider key storage mattered more than model choice.

Read full post + comments ↓

The point of supporting multiple providers isn't to chase whichever model is fastest this month — it's so a researcher can pick based on what they're comfortable sending off-device, session by session. Local Ollama models stay fully offline; cloud providers are opt-in per analysis, with a clear warning before any image leaves the machine.

Persistent session settings and per-provider API key storage turned out to be the unglamorous but essential part of this — nobody wants to re-enter a key every time they switch models.

09 Jun 2026Origins

Why I started building ScriptLens

The original scope, before it became Pandu: a focused tool for studying the geometric shape of historical scripts and how their letterforms change.

Read full post + comments ↓

The starting question was narrow on purpose: could the shape of a letter, described mathematically, tell us something a purely textual comparison couldn't? That led straight to Procrustes analysis, Fourier descriptors, and Hu moments — tools built for shape comparison in completely unrelated fields, repurposed here for palaeography.

ScriptLens began as exactly that: a research tool, not a product. Everything that came after — the AI integration, the databases, the training pipeline — grew out of needing a place to actually store and act on what that geometric analysis found.
